8. Operational Summary
======================

{{- if eq .Values.architecture "standalone" }}
Standalone mode:

  One writable StatefulSet is installed. Use this mode when operational simplicity matters more than read scaling.
{{- else }}
Replication mode:

  One fixed source StatefulSet and one read replica StatefulSet are installed. Replication is asynchronous and does not provide automatic source promotion.
{{- end }}

9. Validation Commands
======================

Check workloads:

  kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}
  kubectl get statefulset,svc,pvc -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}

Check rendered Services:

  kubectl get svc -n {{ .Release.Namespace }} {{ include "mysql.clientServiceName" . }}
{{- if eq .Values.architecture "replication" }}
  kubectl get svc -n {{ .Release.Namespace }} {{ include "mysql.sourceServiceName" . }}
  kubectl get svc -n {{ .Release.Namespace }} {{ include "mysql.replicasServiceName" . }}
{{- end }}

Run a simple SQL smoke test from a temporary client pod:

  kubectl run mysql-client -n {{ .Release.Namespace }} --rm -it --restart=Never --image={{ .Values.image.repository }}:{{ .Values.image.tag }} -- mysql -h {{ include "mysql.clientServiceName" . }} -P {{ .Values.service.port }} -u {{ .Values.auth.username }} -p -e "SELECT 1;"

{{- if eq .Values.architecture "replication" }}
Check replication from a replica pod:

  kubectl exec -n {{ .Release.Namespace }} {{ include "mysql.replicaStatefulSetName" . }}-0 -- mysql -uroot -p -e "SHOW REPLICA STATUS\\G"

{{- end }}
10. Troubleshooting
===================

Initial troubleshooting:

  kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}
  kubectl logs -n {{ .Release.Namespace }} {{ include "mysql.sourceStatefulSetName" . }}-0
{{- if eq .Values.architecture "replication" }}
  kubectl logs -n {{ .Release.Namespace }} {{ include "mysql.replicaStatefulSetName" . }}-0
{{- end }}
  kubectl get events -n {{ .Release.Namespace }} --sort-by=.lastTimestamp

Important:

- replication mode provides one fixed source and asynchronous read replicas
- the chart does not implement automatic source promotion or operator-style HA
- default values are suitable for development; production installs should provide external credentials, persistence, TLS, NetworkPolicy, resources, backups, and operational runbooks
